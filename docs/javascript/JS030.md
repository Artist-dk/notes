## ðŸ”· **1. What is the Fetch API?**  
The **Fetch API** allows you to send HTTP requests to a server and handle responses **asynchronously** using **Promises**.

### âœ… **Why Use Fetch API?**  
âœ” **Promise-based** â†’ No callbacks, easier to manage async code.  
âœ” **Works in modern browsers** â†’ No need for third-party libraries.  
âœ” **Better error handling** â†’ Unlike `XMLHttpRequest`, it only rejects for network failures.

## ðŸ”· **2. Making a Basic `fetch()` Request (GET)**  

### âœ… **Example: Fetching Data from an API**  
```js

  .then(response => response.json())  // Convert response to JSON
  .then(data => console.log(data))   // Handle the response data

```
âœ” **`fetch(url)`** sends a request to the API.  
âœ” **`response.json()`** converts the response to a JavaScript object.  
âœ” **`.then()`** handles the resolved Promise.  
âœ” **`.catch()`** catches errors (e.g., network issues).  

### ðŸ”¹ **Expected Output (Example Response)**  
```json

  "id": 1,
```

## ðŸ”· **3. Handling Responses (`.json()`, `.text()`)**  
The Fetch API returns a `Response` object, which has multiple methods:  
| Method | Description |
|---------|-------------|
| `.json()` | Parses the response as JSON |
| `.text()` | Parses the response as plain text |
| `.blob()` | Handles binary data (images, files) |
| `.formData()` | Handles form data |
| `.arrayBuffer()` | Reads response as an ArrayBuffer |

### âœ… **Example: Fetching Text Data**  
```js
fetch("https://example.com/data.txt")
  .then(response => response.text())  // Convert response to text
  .then(data => console.log(data))   // Print text content
  .catch(error => console.error("Error:", error));
```

## ðŸ”· **4. Error Handling with `.catch()`**
ðŸš¨ **Fetch does NOT reject on HTTP errors (e.g., 404, 500)**. You need to handle them manually.

### âœ… **Example: Checking for Errors**  
```js

  .then(response => {
    if (!response.ok) {  // `ok` is false for 404, 500 errors
      throw new Error(`HTTP Error! Status: ${response.status}`);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error("Fetch Error:", error));
```
âœ” `response.ok` is `false` for 404/500 errors.  
âœ” We **throw an error** manually for non-success responses.

## ðŸ”· **5. Making a POST Request (Sending Data)**  
To send data, use `fetch(url, options)` with:  
- **Method** â†’ `"POST"`  
- **Headers** â†’ `Content-Type: "application/json"`  
- **Body** â†’ `JSON.stringify(data)`

### âœ… **Example: Sending Data with POST**
```js

  title: "New Post",
  body: "This is the content of the new post.",
  userId: 1
};

fetch("https://jsonplaceholder.typicode.com/posts", {
  method: "POST",
  headers: {
    "Content-Type": "application/json"
  },
  body: JSON.stringify(newPost)
})
  .then(response => response.json())  // Convert response to JSON
  .then(data => console.log("Post created:", data))
  .catch(error => console.error("Error:", error));
```
âœ” Data is **stringified** before sending.  
âœ” **Headers** tell the server we're sending JSON data.

## ðŸ”· **7. Making PUT & DELETE Requests**
- **PUT** â†’ Updates existing data.  
- **DELETE** â†’ Removes data.

### âœ… **Example: Updating Data with PUT**
```js

  method: "PUT",
  headers: {

  .then(response => response.json())
  .then(data => console.log("Updated:", data))
  .catch(error => console.error("Error:", error));
```

### âœ… **Example: Deleting Data**
```js

  .then(() => console.log("Post deleted"))
  .catch(error => console.error("Error:", error));
```

## ðŸ”· **8. Using `async/await` with `fetch()`**
Using `async/await` makes Fetch API **cleaner and easier to read**.

### âœ… **Example: Fetching Data with `async/await`**

```js
  } catch (error) {
    console.error("Fetch error:", error);
  }
}

```
âœ” **No more `.then()` chaining**.  
âœ” **Try...catch handles errors elegantly.**
