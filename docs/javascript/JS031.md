## ðŸŒ **2. Geolocation API (Get User Location)**  
The **Geolocation API** allows web apps to access the user's current location.

### âœ… **Check if Geolocation is Supported**  

### âœ… **Getting the User's Current Location**  
Use `navigator.geolocation.getCurrentPosition()` to get latitude & longitude.  
```js

  (position) => {
    console.log("Latitude:", position.coords.latitude);
    console.log("Longitude:", position.coords.longitude);
  },
  (error) => {
    console.error("Error getting location:", error.message);
  }
);
```
âœ” If the user **grants permission**, it returns the coordinates.  
âœ” If the user **denies permission**, an **error** is thrown.  

---

### âœ… **Tracking Userâ€™s Location in Real-Time**  
Use `watchPosition()` to track location continuously.  
```js

  (position) => {

    console.log("Updated Longitude:", position.coords.longitude);
  },
  (error) => {
    console.error("Error:", error.message);
  }

```
âœ” `watchPosition()` **keeps tracking** changes in location.

## ðŸ”” **4. Notification API (System Notifications)**  
The **Notification API** allows web apps to send system notifications.  

### âœ… **Checking Notification Permission**  
```js

  Notification.requestPermission().then(permission => {
    console.log("Permission:", permission);
  });
}
```
âœ” `Notification.permission` returns `"granted"`, `"denied"`, or `"default"`.  
âœ” If permission is `"default"`, you must **request it** using `requestPermission()`.

### âœ… **Sending a Notification**  
```js

    body: "This is a sample notification",
    icon: "https://via.placeholder.com/100"
  });
}
```
âœ” The **title** is required.  
âœ” The **body** contains the message.  
âœ” The **icon** is an optional image.

### âœ… **Triggering a Notification on Button Click**

```html
  if (Notification.permission === "granted") {
    new Notification("New Message!", {

```
âœ” This script asks for **permission** and then **sends a notification**.
